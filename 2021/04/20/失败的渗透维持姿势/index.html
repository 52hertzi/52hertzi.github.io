失败的渗透维持姿势

前言

后渗透一般是指我们成功Getshell后的操作，接下来博主这里以一台Getshell了的windows2012的服务器为例。

注： 本文章出发之意是以学习为目的，且已提醒管理员修补漏洞。

信息收集

查看当前服务器的基础配置信息

这里使用的是冰蝎的shell，可以很清楚的看到当前服务器上一些敏感信息的配置。

查看当前权限

使用命令执行功能

这里可以看到拥有3个用户。

那么接下来看一下当前权限。

好家伙！就很奈斯，不用提权了，直接最高权限

那么接下来就是查看服务器有没有开启远程登陆服务。

当前服务器不仅打开了远程登陆服务（3389），还有Mssql数据库的远程登陆等服务。

那么接下来查看一下远程登陆服务的进程号！

这里看到当前运行了远程登陆服务的进程号为2536，获取了进程号以后，接下来查看运行了远程服务的端口，一般为3389端口。

芜湖！管理员修改了远程登陆端口3389为5366！！！有趣有趣。

测试一下！

OK !!! OK 可以连接

哦豁！完蛋

查看补丁情况，看看是否可以使用神器mimikatz。

注：当安装了KB2871997补丁或者系统版本大于windows server 2012时，系统的内存中就不再保存明文的密码，这样利用mimikatz就不能从内存中读出明文密码了。mimikatz的使用需要administrator用户执行，administrators中的其他用户都不行。

下面将mimikatz神器传到服务器上抓取明文密码，但是一直执行终端失败……难受

尝试直接下载sam文件

可以下载SAM、SYSTEM文件，尝试注册表导出

拒绝执行命令，360又一次给拦截了。

那么尝试使用下载的SAM和SYSTEM文件本地解密

mimikatz启动——————————————————-失败

查询了大量资料后依然读取密码失败

气得我啊啊啊啊啊啊啊啊啊啊啊啊

辣鸡360………..是不是玩不起！！！！

好了，一次失败的经历。
